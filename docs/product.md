# Eigo プロダクト仕様

## 概要

Eigo は、発音練習を通じてリスニング力を伸ばす英語学習 Web アプリである。英検 3 級レベルの語彙を対象に、絵と単語を見ながら発音し、AI がリアルタイムでフィードバックを返す。

保護者が子供のアカウントを作成し、学習状況を確認できる。将来的には講師によるクラス運営にも対応する。

## 背景と課題

英検 3 級のリスニングで聞き取れない学習者は多いが、原因は聴覚ではなく発音にあることが多い。

音声知覚のモーター理論（Liberman, 1967）によれば、人は音声を聞くとき、自分の発声運動系と照合して認識する。正しく発音できない音は、聞いても正しく知覚できない。逆に、正しい発音を身につければリスニング力は向上する。

しかし現状の学習環境では、発音練習は軽視されがちである。リスニング教材は「聞く→解く」のサイクルで、発音の正確さに対するフィードバックが存在しない。独学では自分の発音が正しいか判断する手段がなく、塾や英会話スクールでは講師が一人ひとりの発音を確認する時間がない。

Eigo は、AI による即時の発音評価と音素レベルのフィードバックでこの課題を解決する。

## ロール

3 ロール制。

| ロール | 誰か | できること |
|--------|------|-----------|
| student | 学習者（小学生〜高校生） | 発音練習、自分の進捗確認 |
| instructor | 講師、家庭教師 | クラス管理、課題配信、全生徒の詳細データ閲覧 |
| parent | 保護者 | 子供のアカウント作成、子供の進捗確認 |

### アカウント構造

アカウントの下に学習者プロフィールがぶら下がる構造（Netflix 方式）。アプリ起動時に「練習する人をえらんでね」で学習者を選択する。学習者ごとに練習履歴・苦手音素が分離される。

プロトタイプでは親がアカウントを作成し、子供の学習者プロフィールを作って練習させる。将来的には中高生が自分のアカウントで自分のプロフィールを作ることも想定する。

## ユーザー価値

### 個人利用の学習者

独学では自分の発音が正しいか判断する手段がなく、リスニング教材を繰り返しても聞き取れない音は改善しない。

Eigo は音素レベルのフィードバックにより、苦手な音素を特定し、練習の方向性を与える。

### クラスに所属する生徒

従来のリスニング宿題では正解数しか得られず、聞き取れなかった原因は不明なままだった。発音練習は宿題として成立しにくい。

Eigo では講師が割り当てた単語セットを自分のペースで練習でき、習得状況と苦手音素を自分でも確認できる。

### 保護者

子供の学習状況を把握する手段がなく、「ちゃんとやっているか」「伸びているか」が分からない。

Eigo では子供の練習量・習得状況・苦手ポイントを確認でき、学習の補助に必要な情報を得られる。

### 講師

生徒一人ひとりの発音を確認する時間は限られ、リスニングが苦手な原因の特定は難しい。発音指導は経験と勘に依存しがちである。

Eigo の管理画面では、生徒ごとの苦手音素や割り当ての習得状況をデータで把握できる。

## 機能概要

### 利用形態

プロトタイプでは個人利用のみ。保護者が子供のアカウントを作成し、学習状況を確認できる。

クラス利用（講師によるクラス作成、課題配信、生徒管理）は将来の拡張とする。

### 学習フロー

1. 絵と英単語を表示
2. お手本音声を再生（初回は必須、2回目以降は任意）
3. 学習者が発音
4. AI が音素レベルで評価し、スコア帯に応じたフィードバックを表示
5. 任意の回数繰り返す

回数制限は設けない。

#### フィードバック

スコア（0-100）に応じて段階的にフィードバックを出し分ける。

- **90+**: 正解した音素のハイライト。「かんぺき！」
- **80-89**: 音素の色分け + 間違いの指摘。「上手に言えたね！」
- **60-79**: 音素の色分け + 間違いの指摘 + 発音のコツ。「おしい！ここを直してみよう」
- **0-59**: 音素詳細は非表示。「お手本を聞いてからもう一度言ってみよう」

全スコア帯で正解した音の数（「5この音のうち 4こ 正解」）と、前回からの改善（「前は 3こ → 今は 4こ！」）を表示する。

日本語話者の典型的な混同パターン（r/l, th/s, v/b, f/h）には具体的な発音のコツを提示する。

### モジュール構成

練習はモジュール単位で進める。1 モジュール = 10 単語。各単語に英検 3 級レベルの例文を 2〜3 文用意する。

各単語には「単語練習」と「例文練習」のステップがある。推奨順序は単語 → 例文だが、どのステップからでも自由に挑戦できる。全ステップでスコア 80 以上を達成するとその単語がマスターとなる。10 単語すべてマスターでモジュール完了。

```
モジュール（10 単語）
  ├─ apple
  │   ├─ 単語練習: "apple"          ★★ 85てん
  │   ├─ 例文: "I eat an apple..."   ★  65てん
  │   └─ 例文: "This apple is..."    —  まだ
  │   → 全ステップ 80 以上でマスター
  ...
  → 10 単語全部マスターでモジュール完了
```

#### スコアと判定

- スコア（0-100）は Speechace API の `speechace_score.pronunciation` を使用
- 例文練習ではテキスト全体のスコアに加え、対象単語の `quality_score` も記録する
- マスター判定はビューで動的に算出（score >= 80）。閾値は実ユーザーテストで調整する

### 発音評価

Speechace API の Score Text エンドポイントを使用する。単語・例文ともに音素の正確さ（各音素のスコア、混同パターン、音節ストレス）で評価する。

認識された音素（`sound_most_like`）により「何を何と間違えたか」が分かるため、具体的な矯正指示に変換できる。日本語話者が苦手とする音素（/r/ vs /l/、/θ/ vs /s/、/v/ vs /b/ 等）に基づいた重点フィードバックを提供する。

連結音声（リダクション・リンキング）は API が自動的に許容するため、自然な発音がペナルティになることはない。流暢さ・イントネーションの評価は将来の拡張とする。

日本語話者の発音に対する API の精度はプロトタイプで検証する。

### ロール別の情報表示

データは同じだが、ロールによって見せ方を変える。

#### 生徒向け

小 5-6 に音素スコアの数値や分析を見せても行動に繋がらない。苦手音素の分析結果を自分で読み解く必要がない設計にする。

**練習直後（即時フィードバック）**

- スコア帯に応じたメッセージ（学習フローの「フィードバック」参照）
- 正解した音の数と前回からの改善
- 音素の色分け（60 点以上の場合のみ。低スコア時は非表示）
- 間違えた音への発音のコツ

**モジュール一覧（学習者向け情報の統合）**

学習者ダッシュボードは独立ページにせず、モジュール一覧に統合する。子供は自発的にダッシュボードに遷移しないため、練習の導線上に情報を置く。

- おすすめ単語: 合格に最も近い単語を1つだけ提示
- 全体進捗: マスターした単語数の表示
- 苦手な音ヒント: 正解率 60% 未満の音素を1つだけ表示（データ不足または該当なしなら非表示）

#### 親向け

親が見るのは週 1 回程度。開いて即座に状況が掴めることを優先する。

- **練習の取り組み** — 曜日ドット（月〜日）で今週の練習リズムを表示。「今週 N日 練習した」で継続性を数値化。最終練習日も表示
- **最近の練習** — 直近の練習履歴をモジュール単位でグループ化して表示
- **達成状況** — マスターした単語数、クリアしたモジュール数のスナップショット
- **発音の様子** — 得意な音と苦手な音の両方を表示。音素→文字表記の固定マッピングテーブルで変換（例: /θ/ → /s/ が多い →「"th" の音が苦手です → "s" とまちがえやすい」）
- **お子様の切り替え** — 兄弟がいる場合、子供ごとにダッシュボードを切り替えられる

## スコープ

### プロトタイプで実装する機能

- モジュール制の練習（1 モジュール = 10 単語 + 例文）
- 単語・例文の表示（絵＋テキスト）と音声再生
- マイク録音、Speechace API による発音評価、フィードバック表示
- 合格判定（閾値は仮に 80、実ユーザーテストで調整）
- 個人利用：アカウント作成、自由な練習、練習履歴・苦手音素の確認
- 保護者：子供のアカウント作成、練習量・成長・苦手ポイントの確認、お子様切り替え
- 生徒向け表示：即時フィードバック（色分け・指摘）、おすすめ単語、全体進捗
- 親向け表示：練習の取り組み（曜日ドット）、最近の練習、達成状況、発音の様子（得意・苦手）

### プロトタイプのスコープ外

- クラス利用（クラス作成、生徒登録、課題の割り当て）
- 講師向け管理画面（詳細データ、クラス全体の集計）
- リスニングテスト（絵を選ぶ形式、英検形式）
- 成長の可視化（テストスコア推移、発音×リスニングの相関表示）
- カリキュラムの自動生成・適応的出題
- 流暢さ・イントネーションの評価
- AI によるテスト自動生成
- 個別の発音録音の再生
- 決済・契約管理
- ネイティブアプリ

## ターゲットユーザー

小学 5〜6 年生で英検 3 級に取り組む家庭。親や家庭教師が学習を補助する構図を想定する。小 5-6 は自分で学習を管理できない年齢であり、UI は小学生が一人で迷わず使えるシンプルさを基準に設計する。

中高生や塾・英会話スクールも対象に含めるが、機能・UI の優先度は小 5-6 基準で判断する。

### 購買決定者

保護者、または塾・スクールの講師。

## クラス利用（スコープ外）

講師がクラスを作成し、生徒を招待する。講師は単語セットを課題として配信し、生徒は自分のペースで取り組む。

### 課題の割り当て

講師はクラスに対して単語セットを課題として割り当てることができる。割り当ては推奨範囲であり、生徒は割り当て外の単語も自由に練習できる。

### 講師向け表示

講師は専門知識があるため、音素記号・スコア・混同パターンをそのまま表示する。

- 生徒ごとの練習回数・平均スコアの推移
- 生徒ごとの苦手音素ランキング（混同パターンを含む）
- 課題の割り当てごとの習得状況
- クラス全体の集計（「このクラスは /θ/ が全体的に弱い」等、授業の重点を判断する材料）

## リスニングテスト（スコープ外）

発音練習とは別の機能。練習は「鍛える」、テストは「測る」。

### テスト形式

**絵を選ぶ形式** — 音声を聞いて、正しい絵を 4 択から選ぶ。単語・短い文に対応。小学生でも取り組みやすい。読解力に依存しない。

**英検形式** — 英検 3 級のリスニング問題に近い形式。会話を聞いて質問に答える、文を聞いて適切な応答を選ぶ、など。選択肢はテキスト。中高生の実戦対策になる。

### テスト運用

テストは事前に用意する。テストセットのリストから選んで受ける形式。講師や親が指定してもよいし、学習者が自分で選んでもよい。やり直しなしの 1 回勝負で、結果を記録する。

テストセットにはレベルを設定する。同レベル内でスコアを比較することで、リスニング力の推移を正しく測定できる。

### AI 生成（将来機能）

将来的には AI がテスト音声（TTS）と問題文を自動生成し、テストを量産できるようにする。

## 成長の可視化（スコープ外）

「発音練習でリスニング力が伸びた」を示す仕組み。

### テストスコアの推移

同レベルのリスニングテストを定期的に受け、スコアの推移を記録する。親にとって最も説得力があるのは「3 ヶ月前は 60 点、今月は 80 点」のような折れ線グラフ。これが継続利用の動機になる。

### 発音×リスニングの相関

発音練習のスコア推移とリスニングテストのスコア推移を並べて表示する。「発音が上達した時期にリスニングも上がった」という相関が可視化されれば、Eigo の仮説（発音→リスニング）の証拠になる。親や講師にとって「なぜ発音練習をやるのか」の納得材料。

### テスト受験の動機づけ

生徒が自発的にテストを受けるとは限らない。親や講師が「月に 1 回テストを受けよう」と促す、あるいはアプリが一定量の練習後に「力試しをしてみよう」とテストを提案する。

## Open Issues

- 大人向け展開：より高難度の語彙を収録した大人向けプロダクトの可能性。対象語彙・UI の差分は未定。
- フラッピング評価：Speechace API がリダクション・リンキングは自動対応するが、フラッピング（/t/ → [ɾ]）への対応は未文書化。実機検証が必要。
- 合格閾値：仮に 80（Very Good 以上）としているが、小 5-6 の実音声で適切か検証が必要。
- Safari 音声録音：MediaRecorder API の Safari 対応状況を検証し、必要に応じて polyfill やライブラリを導入する。
- お手本音声：プロトタイプではブラウザの SpeechSynthesis API（音声合成）を使用。音質は OS・ブラウザ依存でネイティブ音声とは程遠い。本番ではプロ TTS サービス（Google Cloud TTS、OpenAI TTS 等）またはネイティブ録音に差し替える。
