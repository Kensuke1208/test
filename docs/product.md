# Eigo プロダクト仕様

## 概要

Eigo は、発音練習を通じてリスニング力を伸ばす英語学習 Web アプリである。英検 3 級レベルの語彙を対象に、絵と単語を見ながら発音し、AI がリアルタイムでフィードバックを返す。

個人利用とクラス利用の 2 つの形態がある。個人利用ではクラスに所属せず自由に練習できる。クラス利用では、講師が生徒の学習状況を確認できる管理画面が加わる。保護者は子供のアカウントを作成し、学習状況を確認できる。

## 背景と課題

英検 3 級のリスニングで聞き取れない学習者は多いが、原因は聴覚ではなく発音にあることが多い。

音声知覚のモーター理論（Liberman, 1967）によれば、人は音声を聞くとき、自分の発声運動系と照合して認識する。正しく発音できない音は、聞いても正しく知覚できない。逆に、正しい発音を身につければリスニング力は向上する。

しかし現状の学習環境では、発音練習は軽視されがちである。リスニング教材は「聞く→解く」のサイクルで、発音の正確さに対するフィードバックが存在しない。独学では自分の発音が正しいか判断する手段がなく、塾や英会話スクールでは講師が一人ひとりの発音を確認する時間がない。

Eigo は、AI による即時の発音評価と音素レベルのフィードバックでこの課題を解決する。

## ロール

3 ロール制。

| ロール | 誰か | できること |
|--------|------|-----------|
| student | 学習者（小学生〜高校生） | 発音練習、自分の進捗確認 |
| instructor | 講師、家庭教師 | クラス管理、課題配信、全生徒の詳細データ閲覧 |
| parent | 保護者 | 子供のアカウント作成、子供の進捗確認 |

### 親子の紐付け

親がまず自分のアカウントを作成し、そこから子供のアカウントを作成する。紐付けは作成時に自動で行われる。1 人の親が複数の子供を持てる（兄弟対応）。

親が子供のデータを見る経路は、講師がクラス経由で見る経路とは独立している。子供がクラスに所属していなくても（個人利用でも）、親はデータを閲覧できる。

## ユーザー価値

### 個人利用の学習者

独学では自分の発音が正しいか判断する手段がなく、リスニング教材を繰り返しても聞き取れない音は改善しない。

Eigo は音素レベルのフィードバックにより、苦手な音素を特定し、練習の方向性を与える。

### クラスに所属する生徒

従来のリスニング宿題では正解数しか得られず、聞き取れなかった原因は不明なままだった。発音練習は宿題として成立しにくい。

Eigo では講師が割り当てた単語セットを自分のペースで練習でき、習得状況と苦手音素を自分でも確認できる。

### 保護者

子供の学習状況を把握する手段がなく、「ちゃんとやっているか」「伸びているか」が分からない。

Eigo では子供の練習量・習得状況・苦手ポイントを確認でき、学習の補助に必要な情報を得られる。

### 講師

生徒一人ひとりの発音を確認する時間は限られ、リスニングが苦手な原因の特定は難しい。発音指導は経験と勘に依存しがちである。

Eigo の管理画面では、生徒ごとの苦手音素や割り当ての習得状況をデータで把握できる。

## 機能概要

### 利用形態

**個人利用**: アカウントを作成し、英検 3 級の語彙から自由に練習する。クラスへの所属は不要。練習履歴、苦手音素、習得状況を確認できる。

**クラス利用**: 講師がクラスを作成し、生徒を招待する。講師は単語セットを課題として配信し、生徒は自分のペースで取り組む。講師は管理画面で各生徒の進捗と苦手音素を確認できる。

### 学習フロー

1. 絵と英単語を表示
2. ネイティブ音声を再生
3. 学習者が発音
4. AI が音素レベルで評価し、フィードバックを表示（正しい音素は緑、誤りは赤、改善ポイントを提示）
5. 任意の回数繰り返す

スコアが 80 以上に達すると「習得済み」となる。回数制限は設けない。

### モジュール構成

練習はモジュール単位で進める。1 モジュール = 10 単語。各単語に英検 3 級レベルの例文を 2〜3 文用意する。

単語ごとに「単語練習 → 例文練習」の順で取り組み、例文までクリアするとその単語が合格となる。10 単語すべて合格でモジュール合格。

```
モジュール（10 単語）
  ├─ apple
  │   ├─ 単語練習: "apple"
  │   ├─ 例文: "I eat an apple every morning."
  │   └─ 例文: "This apple is very sweet."
  │   → 全部クリアで apple 合格
  ├─ river
  │   ├─ 単語練習: "river"
  │   ├─ 例文: ...
  │   └─ 例文: ...
  │   → 全部クリアで river 合格
  ...
  → 10 単語全部合格でモジュール合格
```

例文では音素評価に加えて、連結・脱落・リズム（プロソディ）の評価が加わる。Speechace API の Score Text エンドポイントで単語・例文の両方に対応可能。

### 課題の割り当て（クラス利用）

講師はクラスに対して単語セットを課題として割り当てることができる。割り当ては推奨範囲であり、生徒は割り当て外の単語も自由に練習できる。講師は管理画面で割り当てごとの習得状況を確認できる。

### 発音評価

Speechace API の Score Text エンドポイントを使用する。音素・音節・単語レベルのスコアリングに加え、認識された音素（sound_most_like）を返すため、具体的な矯正指示に変換できる。

日本語話者が苦手とする音素（/r/ vs /l/、/θ/ vs /s/、/v/ vs /b/ 等）に基づいた重点フィードバックを提供する。日本語話者の発音に対する API の精度はプロトタイプで検証する。

### ロール別の情報表示

データは同じだが、ロールによって見せ方を変える。

#### 生徒向け：次にやることを見せる

小 5-6 に音素スコアの数値や分析を見せても行動に繋がらない。ダッシュボードではなく、次の練習への導線を提供する。

- 習得進捗：「120 / 300 単語マスター」のようなプログレスバー
- 練習の即時フィードバック：上手く言えた・言えなかったの色分け（学習フローに含まれる）
- 練習推薦：「この音をもう少し練習しよう」と、苦手音素を含む単語を提示する

苦手音素の分析結果を自分で読み解く必要がない設計。

#### 親向け：5 秒で掴める 3 指標

親が見るのは週 1 回程度。開いて即座に状況が掴めることを優先する。

- **練習量** — 今週の練習回数、最後に練習した日（「やっているか」）
- **成長** — 習得単語数の推移（「伸びているか」）
- **苦手ポイント** — 「"th" と "r" の音が苦手です」程度の自然言語サマリー。音素記号ではなく具体的な文字表記を使う

#### 講師向け：詳細データ

講師は専門知識があるため、音素記号・スコア・混同パターンをそのまま表示する。

- 生徒ごとの練習回数・平均スコアの推移
- 生徒ごとの苦手音素ランキング（混同パターンを含む）
- 課題の割り当てごとの習得状況
- クラス全体の集計（「このクラスは /θ/ が全体的に弱い」等、授業の重点を判断する材料）

個別の発音録音の再生機能は、プロトタイプのスコープ外とする。

## スコープ

### プロトタイプで実装する機能

- モジュール制の練習（1 モジュール = 10 単語 + 例文）
- 単語・例文の表示（絵＋テキスト）と音声再生
- マイク録音、Speechace API による発音評価、フィードバック表示
- スコア 80 以上で習得済み判定
- 個人利用：アカウント作成、自由な練習、練習履歴・苦手音素の確認
- クラス利用：クラス作成、生徒登録、単語セット割り当て
- 保護者：子供のアカウント作成、練習量・成長・苦手ポイントの確認
- 生徒向け表示：プログレスバー、練習推薦
- 親向け表示：練習量・成長・苦手ポイントの 3 指標

### プロトタイプのスコープ外

- リスニングテスト（絵を選ぶ形式、英検形式）
- 成長の可視化（テストスコア推移、発音×リスニングの相関表示）
- 講師向け管理画面（詳細データ、クラス全体の集計）
- カリキュラムの自動生成・適応的出題
- AI によるテスト自動生成
- 決済・契約管理
- ネイティブアプリ

## ターゲットユーザー

小学 5〜6 年生で英検 3 級に取り組む家庭。親や家庭教師が学習を補助する構図を想定する。小 5-6 は自分で学習を管理できない年齢であり、UI は小学生が一人で迷わず使えるシンプルさを基準に設計する。

中高生や塾・英会話スクールも対象に含めるが、機能・UI の優先度は小 5-6 基準で判断する。

### 購買決定者

保護者、または塾・スクールの講師。

## リスニングテスト（スコープ外）

発音練習とは別の機能。練習は「鍛える」、テストは「測る」。

### テスト形式

**絵を選ぶ形式** — 音声を聞いて、正しい絵を 4 択から選ぶ。単語・短い文に対応。小学生でも取り組みやすい。読解力に依存しない。

**英検形式** — 英検 3 級のリスニング問題に近い形式。会話を聞いて質問に答える、文を聞いて適切な応答を選ぶ、など。選択肢はテキスト。中高生の実戦対策になる。

### テスト運用

テストは事前に用意する。テストセットのリストから選んで受ける形式。講師や親が指定してもよいし、学習者が自分で選んでもよい。やり直しなしの 1 回勝負で、結果を記録する。

テストセットにはレベルを設定する。同レベル内でスコアを比較することで、リスニング力の推移を正しく測定できる。

### AI 生成（将来機能）

将来的には AI がテスト音声（TTS）と問題文を自動生成し、テストを量産できるようにする。

## 成長の可視化（スコープ外）

「発音練習でリスニング力が伸びた」を示す仕組み。

### テストスコアの推移

同レベルのリスニングテストを定期的に受け、スコアの推移を記録する。親にとって最も説得力があるのは「3 ヶ月前は 60 点、今月は 80 点」のような折れ線グラフ。これが継続利用の動機になる。

### 発音×リスニングの相関

発音練習のスコア推移とリスニングテストのスコア推移を並べて表示する。「発音が上達した時期にリスニングも上がった」という相関が可視化されれば、Eigo の仮説（発音→リスニング）の証拠になる。親や講師にとって「なぜ発音練習をやるのか」の納得材料。

### テスト受験の動機づけ

生徒が自発的にテストを受けるとは限らない。親や講師が「月に 1 回テストを受けよう」と促す、あるいはアプリが一定量の練習後に「力試しをしてみよう」とテストを提案する。
